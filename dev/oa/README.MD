# OA 基座开发说明

依据：`SAD.md` §3、`Permission-Design.md` §1-§3。

## 首期目标

- 提供 OA 权限域与菜单入口。
- 复用 IAM 的用户、角色、权限点能力。
- 支持按角色分配 OA 权限，并允许对单个用户做权限加授或撤销。

## 开发顺序

1. IAM 权限模型落库。
2. OA 权限点初始化。
3. Web 管理端接入权限菜单。
4. 权限变更审计。

## 第一阶段实现清单

依据：`plans/oa20260623.plan.md`。

- 流程定义、节点、节点处理人。
- 入库、出库、报销三个内置流程。
- USER / ROLE / SUPERVISOR 三种处理模式。
- 待办、审批、驳回、催办、自动提醒日志。
- 企业微信机器人配置与消息发送日志。
- OA 审批通过后触发物资入库/出库/报销业务服务。
