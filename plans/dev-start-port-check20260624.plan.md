# 本地启动脚本端口预检计划

## 目标

- 在启动前检查关键端口是否被占用。
- 避免后端、前端、PostgreSQL 或 Redis 未实际启动却被误认为启动成功。
- 输出占用端口的进程信息，方便开发人员处理。

## 范围

- 修改 Linux/macOS 启动脚本。
- 修改 Windows PowerShell 启动脚本。
- 更新开发文档中的端口冲突说明。
- 不修改业务代码、API、数据库结构或权限模型。

## 端口范围

- 后端：`18080`
- Web 管理端：`15173`
- PostgreSQL：`5432`
- Redis：`6379`

## 执行顺序

1. 检查当前脚本启动流程和端口配置来源。
2. 在启动基础设施前检查 PostgreSQL/Redis 端口，除非传入 `--no-infra` / `-NoInfra`。
3. 在启动前后端前检查后端和前端端口。
4. 端口被占用时输出占用进程并退出。
5. 更新文档中的处理说明。
6. 验证脚本语法和端口检测行为。

## 验收标准

- 端口被占用时，脚本拒绝继续启动并输出占用信息。
- `--no-infra` 时不检查 PostgreSQL/Redis 端口。
- 文档说明默认端口、检测策略和解决方式。
