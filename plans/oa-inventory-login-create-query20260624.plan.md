# OA 登录与物资管理基础能力补齐计划

## 需求分析

1. OA/Web 管理端必须先登录后使用，提供登录界面。
2. 管理员用户初始密码为 `1qaz@WSX`。
3. 其他成员初始密码为 `qwer1234`。
4. 物资档案支持新增。
5. 库存支持新增。
6. 库存查询支持模糊查询。
7. 物资档案 ID 与库存 ID 支持数据库自增。

## 业务分析

- 登录是 OA 基座访问前置条件，未登录用户不得进入管理端业务页面。
- 初始密码规则只用于种子账号或新建成员初始化，不代表长期密码策略。
- 物资档案是库存的主数据；库存新增应关联物资档案与仓库/位置等库存维度。
- 模糊查询用于按物资编码、名称、分类、仓库等快速检索库存。

## 数据分析

- 检查现有数据字典、DDL 与实体模型。
- 若当前 ID 不是自增，改为 PostgreSQL identity/bigserial 对应方式。
- 若缺少用户种子数据、密码字段或物资/库存新增所需字段，先补设计文档，再补 DDL/代码。

## 权限分析

- 登录接口匿名可访问。
- 物资档案与库存查询/新增需要登录用户权限。
- 管理员默认拥有 OA 与物资管理权限。
- 权限审计若已有基础结构则接入；若当前阶段未实现完整审计，记录为风险。

## AI 分析

- 登录接口不允许 AI 调用。
- 物资档案/库存新增不允许 AI 直接调用写入。
- 库存查询可作为后续 AI 只读候选接口，但本轮若未登记 MCP 白名单则仍不开放给 AI。

## 系统设计

- 后端补登录认证接口与会话/JWT 或简化令牌机制，按现有项目能力选择最小可用实现。
- 前端补登录页面与路由守卫。
- 后端补物资档案新增、库存新增、库存模糊查询接口。
- 前端补对应新增表单和查询输入。

## 风险

- 当前项目可能尚未实现完整 RBAC/JWT，本轮以最小可用登录闭环为目标。
- 初始密码写入种子数据需要散列存储，不能明文落库。
- 若现有数据库 volume 已初始化，DDL 改动需要重建数据库或补迁移脚本。
