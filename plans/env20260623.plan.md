# 环境配置与连通性执行计划

日期：2026-06-23

## 目标

补齐当前开发环境缺失项，优先使用直连网络完成依赖安装、构建与测试；仅当直连不可用时再配置代理。

## 范围

- 检查系统代理环境变量、Git/npm/Maven/Docker 相关代理配置。
- 检查 GitHub、npm registry、Maven Central、Docker registry、Ubuntu apt 源连通性。
- 补齐 Maven、Docker CLI/Compose、前端依赖等缺失环境。
- 运行后端测试、前端构建、Docker Compose 配置校验。

## 执行顺序

1. 检查当前代理配置和系统环境。
2. 直连测试外部依赖源。
3. 若直连可用，清理项目级代理配置并直接安装依赖。
4. 若直连不可用，查找可用代理配置后再重试。
5. 验证 Maven、npm、Docker、Compose。
6. 运行项目构建和测试。

## 验收标准

- 明确当前是否走代理。
- Maven 可用并能执行后端测试。
- npm 依赖安装完成，Web 管理端可构建。
- Docker CLI 与 Compose 可用，`docker compose config` 通过。
- 记录无法完成项及原因。

## 风险

- Docker 服务可能受宿主机权限或 systemd 状态限制。
- npm/Maven/Docker 依赖源可能受网络波动影响。
- 若需要代理但没有可用代理地址，只能记录并等待用户提供。
