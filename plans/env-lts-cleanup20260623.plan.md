# 环境代理清理与 LTS 版本升级计划

## 目标

- 清理已经无效的旧代理端口 `17890` 配置，保持项目默认直连。
- 当前环境仍优先直连；只有官方源不可达时才临时启用可用代理。
- 核查前端、后端框架版本，并升级到当前官方 LTS 或稳定推荐线。
- 完成依赖安装、后端测试、前端构建与基础配置验证。

## 执行顺序

1. 检查 shell、Git、npm、项目配置中是否仍残留 `17890`。
2. 检查当前前后端版本，包括 Java、Spring Boot、Node、Vue、Vite、Element Plus。
3. 查询官方版本来源，确认当前最新 LTS/稳定推荐线。
4. 清理失效代理配置，保留直连默认策略。
5. 更新项目版本配置与锁文件。
6. 运行后端测试、前端构建、npm 审计和 Docker Compose 配置检查。

## 验收标准

- `17890` 不再出现在生效环境配置中。
- 新 shell 默认不设置代理变量。
- 后端框架版本与 Java LTS 版本一致或满足当前官方支持策略。
- 前端框架与构建工具升级到当前稳定推荐版本线。
- `mvn test`、`npm run build`、`docker compose config` 通过。

## 风险控制

- 不修改用户未要求的业务逻辑。
- 如最新 LTS 版本与现有框架生态不兼容，优先记录原因并选择官方支持的最高兼容版本。
- 所有框架升级后的行为变化需通过测试与构建验证。
