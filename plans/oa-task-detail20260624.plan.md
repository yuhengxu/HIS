# OA 待办详情修复计划（2026-06-24）

## 1. 需求分析

- manager 登录后能看到待办，但无法点进待办查看 OA 申请内容。
- 待办列表需要增加“详情”入口。
- 详情页/弹窗需要展示 OA 申请表单内容，审批人据此判断同意或驳回。

## 2. 业务分析

- 审批人处理流程前必须能看到发起人、流程类型、当前节点、申请内容。
- 入库、领用、报销等 OA 的表单内容需要以可读方式展示。

## 3. 数据分析

- 本轮不新增表。
- 复用 `oa_task.process_instance_id` 关联 `oa_process_instance.form_data`。
- 物资与仓库基础名称用于详情展示，不改变业务数据。

## 4. 权限分析

- 待办详情接口需要 `oa:task:read`。
- 只有当前可处理该待办的用户可以查看该待办详情。
- AI 不允许调用待办详情接口。

## 5. AI 分析

- 待办详情包含个人申请和业务表单数据，不进入 AI 白名单。
- AI 不获得审批详情读权限，也不参与审批写操作。

## 6. 系统设计

- 后端新增待办详情 DTO 和接口。
- 前端工作台待办列表增加“详情”按钮。
- 详情使用抽屉展示申请内容，并支持在详情中审批/驳回。

## 7. 开发设计

- 新增 `OaTaskDetailRecord`。
- `ProcessRuntimeService.taskDetail` 校验权限和处理人范围。
- `OaTaskController` 新增 `GET /api/v1/oa/tasks/{taskId}`。
- `InventoryStockService` 提供 OA 展示用的物资/仓库名称解析。
- `Dashboard.vue` 增加详情抽屉与表单内容展示。
- 更新 API 文档与测试。
