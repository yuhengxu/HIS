# 后端包结构品牌中性修正计划

## 目标

- 全仓不保留刚才误引入的品牌词。
- 后端 Java 包结构改为品牌中性命名：
  - `com.health.platform`：共享平台能力，包括 IAM、认证、审计、AI 接入、网关与公共基础设施。
  - `com.health.his`：HIS 医院信息系统独立业务域。
  - `com.health.kangyang`：康养服务系统独立业务域。
- HIS 与康养保持并列独立，不互相包含。

## 范围

- 后端源码包名与物理路径。
- 后端测试包名与 import。
- 文档、计划和开发说明中的包结构描述。
- 不修改 API 路径、数据库表名、权限点和业务行为。

## 执行顺序

1. 将刚才误引入的品牌包路径迁移到 `com/health`。
2. 替换源码、测试、文档、计划里的品牌包名残留。
3. 调整应用启动类包扫描到 `com.health`。
4. 更新架构文档和开发文档中的模块边界说明。
5. 运行后端测试。
6. 执行全仓残留扫描，确认没有品牌词残留。

## 验收标准

- 全仓搜索不到刚才误引入的品牌词。
- 后端共享能力位于 `com.health.platform`。
- HIS 与康养独立业务域分别位于 `com.health.his` 与 `com.health.kangyang`。
- `mvn test` 通过。

## 风险

- 本轮只修正命名与边界，不拆分 Maven 多模块。
- 包名调整会影响所有 Java import，需要通过测试验证。
