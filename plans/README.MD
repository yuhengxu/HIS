# plans · 执行计划

本目录保存每轮结构性请求的执行计划，是 README.MD 宪法中“先计划后执行”的落点。

## 命名

建议使用：`YYYYMMDD-topic.plan.md`。

## 每份计划至少包含

- 目标与范围
- 不做什么
- 需求分析、业务分析、数据分析、权限分析、AI 分析
- 系统设计与开发设计
- 涉及文档、代码、测试清单
- 执行顺序
- 验收标准与风险清单

## 执行规则

1. 每轮请求先生成或引用计划。
2. 按计划顺序推进：design/dev 文档 -> 数据库/API/权限 -> 代码 -> 测试。
3. 若执行中发现计划缺失或偏离，先更新计划或记录偏离原因。
4. 完成后在最终回复中说明计划执行结果和剩余风险。
