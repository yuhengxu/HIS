# 界面中文化、部门负责人物资档案与菜单配置生效计划（2026-06-24）

## 1. 需求分析

- 界面展示内容必须使用中文，不能直接展示英文枚举值。
- 部门负责人需要能查看物资档案，用于审批时理解物资信息。
- 角色菜单配置删除“我的待办”后，左侧菜单不应继续显示旧入口。

## 2. 业务分析

- 工作台已经承载待办处理，“我的待办”独立菜单可从默认菜单中移除。
- 部门负责人作为审批人需要读物资档案，但不应获得物资档案写权限。
- 菜单配置是系统管理能力，前端左侧菜单必须以服务端当前用户菜单为准。

## 3. 数据分析

- 本轮不新增数据库表。
- 调整初始化角色权限和菜单种子数据。
- 前端使用 `/api/v1/me/menus` 的菜单树作为渲染来源。

## 4. 权限分析

- `DEPARTMENT_MANAGER` 增加 `inventory:item:read`，仅可读物资档案。
- “我的待办”独立菜单不再默认创建，待办保留在工作台内。
- 接口权限仍是最终边界，菜单只控制入口。

## 5. AI 分析

- 本轮不新增 AI 可调用接口。
- 物资档案读取仍按现有只读接口，AI 边界不变。

## 6. 系统设计
