# 产品需求文档（PRD）

## 1. 首期范围

依据：`README.MD` §3、`SAD.md` §2-§3。

首期从 OA 基座和物资管理模块开始，建立统一账户、基础权限、审计、AI 只读接入和一键部署能力，为后续 HIS 与康养业务模块提供基础设施。

## 2. 用户管理与 OA 权限

### 2.1 目标

- 支持统一用户管理。
- 引入角色概念，根据角色划分基础 OA 访问权限。
- 支持按单个成员额外开启或撤销某个 OA/物资权限。
- 所有权限变更必须可审计。

### 2.2 角色

| 角色 | 说明 |
|---|---|
| 系统管理员 | 管理用户、角色、权限、审计配置 |
| OA 管理员 | 管理 OA 访问权限与后续办公能力 |
| 物资管理员 | 管理物资档案、入库、出库、盘点 |
| 普通员工 | 根据角色和单独授权访问 OA/物资功能 |
| AI 调用方 | 仅可调用 MCP 白名单中的只读接口 |

### 2.3 验收标准

- Given 用户拥有角色权限 When 访问对应 OA 菜单 Then 允许访问。
- Given 用户没有角色权限但被单独加授 When 访问对应功能 Then 允许访问。
- Given 用户被单独撤销权限 When 访问对应功能 Then 拒绝访问。
- Given 管理员调整用户权限 When 保存成功 Then 记录权限变更审计日志。

## 3. 物资管理模块

### 3.1 目标

覆盖物资分类、物资档案、供应商、仓库、库存、入库、出库、盘点、库存流水。

### 3.2 验收标准

- Given 物资管理员创建物资档案 When 提交必填信息 Then 系统保存物资并可查询。
- Given 入库单审核通过 When 生成库存流水 Then 库存余额增加。
- Given 出库单审核通过 When 生成库存流水 Then 库存余额减少且不得低于业务允许范围。
- Given 盘点完成 When 存在账实差异 Then 生成盘点流水和审计记录。
- Given AI 调用库存查询接口 When scope 合法 Then 仅返回只读库存数据。

## 4. 非功能需求

- 支持 Windows/Linux、局域网/云端部署。
- 首期使用 Docker Compose 一键启动。
- Web 管理端适配低配电脑，移动端优先微信小程序和企业微信小程序。
- AI 数据库账号只读，接口调用受 MCP 白名单限制。
