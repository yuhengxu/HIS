# 产品需求文档（PRD）

## 1. 首期范围

依据：`README.MD` §3、`SAD.md` §2-§3。

首期从 OA 基座和物资管理模块开始，建立统一账户、基础权限、审计、AI 只读接入和一键部署能力，为后续 HIS 与康养业务模块提供基础设施。

## 2. 用户管理与 OA 权限

### 2.1 目标

- 支持统一用户管理。
- 引入角色概念，根据角色划分基础 OA 访问权限。
- 支持按单个成员额外开启或撤销某个 OA/物资权限。
- 所有权限变更必须可审计。

### 2.2 角色

| 角色 | 说明 |
|---|---|
| 系统管理员 | 管理用户、角色、权限、审计配置 |
| OA 管理员 | 管理 OA 访问权限与后续办公能力 |
| 物资管理员 | 管理物资档案、入库、出库、盘点 |
| 普通员工 | 根据角色和单独授权访问 OA/物资功能 |
| AI 调用方 | 仅可调用 MCP 白名单中的只读接口 |

### 2.3 验收标准

- Given 用户拥有角色权限 When 访问对应 OA 菜单 Then 允许访问。
- Given 用户没有角色权限但被单独加授 When 访问对应功能 Then 允许访问。
- Given 用户被单独撤销权限 When 访问对应功能 Then 拒绝访问。
- Given 管理员调整用户权限 When 保存成功 Then 记录权限变更审计日志。

## 3. 物资管理模块

### 3.1 目标

覆盖物资分类、物资档案、供应商、仓库、库存、入库、出库、盘点、库存流水。

### 3.2 验收标准

- Given 物资管理员创建物资档案 When 提交必填信息 Then 系统保存物资并可查询。
- Given 入库单审核通过 When 生成库存流水 Then 库存余额增加。
- Given 出库单审核通过 When 生成库存流水 Then 库存余额减少且不得低于业务允许范围。
- Given 盘点完成 When 存在账实差异 Then 生成盘点流水和审计记录。
- Given AI 调用库存查询接口 When scope 合法 Then 仅返回只读库存数据。

## 4. 非功能需求

- 支持 Windows/Linux、局域网/云端部署。
- 首期使用 Docker Compose 一键启动。
- Web 管理端适配低配电脑，移动端优先微信小程序和企业微信小程序。
- AI 数据库账号只读，接口调用受 MCP 白名单限制。

## 5. OA 流程管理

依据：`plans/oa20260623.plan.md` §3、§6、§13。

### 5.1 目标

- 支持可配置 OA 流程定义、流程节点和节点处理人。
- 节点处理人支持指定用户、指定角色、汇报上级三种模式。
- 内置物资入库 OA、物资出库 OA、报销 OA。
- 入库/出库必须通过 OA 审批后才影响库存。

### 5.2 验收标准

- 管理员可配置流程节点顺序、节点类型、处理模式和提醒策略。
- 指定处理人节点只能本人审批。
- 角色处理节点由拥有该角色的任意 enabled 用户审批即可流转。
- 汇报上级节点自动解析发起人的 `report_to_user_id`；缺失时进入配置异常，不跳过审批。
- 流程支持发起、待办、审批、驳回、催办、自动提醒日志。

## 6. 企业微信通知与催办

### 6.1 目标

- 支持企业微信机器人配置，不在代码中硬编码 Webhook。
- 任务到达、自动提醒、手动催办均记录消息日志。
- 发送失败不阻断 OA 主流程。

### 6.2 验收标准

- OA 任务到达处理人后生成 `task_arrived` 消息日志。
- 发起人可催办，系统按冷却时间限制重复催办。
- 自动提醒按流程或节点配置频率执行。

## 7. 物资入库/出库 OA 关联

- 入库 OA 审批通过后生成入库单、入库明细、库存流水并增加库存。
- 出库 OA 审批通过后生成出库单、出库明细、库存流水并扣减库存。
- 出库扣减不得导致库存为负。

## 8. 报销 OA 与入库 OA 关联

- 报销 OA 可选择关联一个已审批通过的入库 OA。
- 报销审批通过后生成报销记录，不直接变更库存。

## 9. 物资库类型与价格管理

- 仓库分为医疗用品库和非医疗用品库。
- 物资支持医疗/非医疗类型、默认价、最新价。
- 价格记录支持采购价、报销价、参考价。

## 10. 用户管理增强

依据：`plans/oa20260624-iam-oa-inventory-enhancement.plan.md` §5。

- 系统管理员可新建用户，默认初始密码 `qwer1234`，数据库仅保存密码哈希。
- 支持设置角色、汇报上级、企业微信 ID、启停用、重置密码、软删除。
- Given 系统管理员创建用户 When 保存成功 Then 新用户初始密码为 `qwer1234`。
- Given 管理员重置密码 When 操作成功 Then 密码重置为 `qwer1234` 并记录审计。

## 11. 角色权限维护增强

- 系统管理员可新增、修改、删除（软删除）或禁用非系统内置角色。
- 每个权限点在前端和接口返回中包含中文名称与说明。
- Given 系统管理员新增角色 When 勾选权限点并保存 Then 角色权限绑定生效并记录审计。
- Given 角色已绑定用户 When 尝试删除 Then 系统拒绝硬删。

## 12. OA 流程发起增强

- 业务用户可在 OA 发起页选择发起物资入库、物资出库、报销流程实例。
- Given 用户拥有 `oa:instance:create` When 打开发起页 Then 可选择 `MATERIAL_INBOUND`、`MATERIAL_OUTBOUND`、`REIMBURSEMENT`。
- Given 用户提交报销 OA When 未上传报销凭证 Then 系统拒绝提交。

## 13. OA 与物资档案联动

- OA 表单支持搜索并选择已有物资；未找到时可录入新物资信息。
- 非物资管理员录入的新物资先保存为 OA 内物资草稿，审批通过后创建正式物资档案。
- Given 物资管理员在 OA 中录入新物资 When 提交 Then 可即时创建正式物资档案。

## 14. 物资图片与报销凭证上传

- 通用附件能力支持物资图片、OA 附件、报销凭证上传，校验类型、大小与权限。
- 物资档案写操作与物资图片维护仅 `SYSTEM_ADMIN`、`INVENTORY_ADMIN` 可执行。
- Given 报销 OA 未上传凭证 When 提交 Then 不可提交。
- Given 普通员工访问物资档案 When 仅拥有读权限 Then 不显示新增/维护按钮。

## 15. OA 流程定义维护增强

- 系统管理员/OA 管理员可新增、修改、删除/停用流程定义，支持发布与复制。
- 已产生实例的流程不可物理删除，只能停用或软删除。
- 已发布且产生实例的流程修改内容时须生成新版本。
- Given OA 管理员新增流程 When 保存并发布 Then 用户可在发起页选择该流程。

## 16. OA 流程节点灵活配置

- 支持新增、修改、删除、排序审批节点；处理人模式含指定用户、角色、汇报上级、发起人自选。
- 节点为空或处理人无效时不可发布。
- Given 流程已有历史实例 When 修改节点 Then 生成新版本且历史实例仍用旧版本。

## 17. 菜单管理

- 系统管理员可维护菜单树，并按角色、用户配置可见菜单。
- 最终可见菜单 = 角色菜单 + 用户加授 - 用户撤销；菜单可见性不等于接口权限。
- Given 系统管理员配置角色菜单 When 用户重新登录 Then 只返回最终可见菜单树。

## 18. 物资档案与库存查询图片上传补齐

- 物资档案支持主图与多图上传、预览、删除、设主图；库存查询展示物资主图并支持库存现场图。
- 仅 `SYSTEM_ADMIN`、`INVENTORY_ADMIN` 可上传/删除物资与库存图片。
- Given 物资管理员上传主图 When 保存成功 Then 列表与详情展示缩略图。
